## 3. Inferno defects found + fixed (all real, all committed + regression-tested)
1. CapabilityStatement missing `instantiates` + noisy `format` → fixed.
2. **POST `_search`** unimplemented (US Core `_id` test POSTs it) → added shared search executor.
3. **Restart durability** — table registration was in-memory; a restarted server couldn't read its
   own data → `registerExistingTables()` on startup.
4. **Choice-type `[x]`** required-element check false-rejected valid `medication[x]`/`value[x]`.
5. **Bare-id reference search** — index stored `Patient/123`, only exact-matched → clinical
   patient-scoped searches returned 0.
6. **Conditional references** (`Type?identifier=…`) persisted literally (non-conformant) → resolve
   to `Type/<id>` in the transaction handler; `ifNoneExist` conditional create.
7. **Terminology endpoints** never exposed → built the tx-server operations.

Env fixes (not our code): validator OOM → `-Xmx5g` + `SESSION_CACHE_DURATION=10` (see §4).
